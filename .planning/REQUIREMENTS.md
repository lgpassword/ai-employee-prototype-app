# 需求边界

## P0 必做

- 数据持久化：先落地本地 JSON/SQLite 级别的持久化入口，再迁移到正式数据库。
- 登录与权限安全：替换明文密码和全局 session，补充角色、额度和租户边界。
- 任务中心：支持 AI 任务创建、拆解、状态、时间线、工具调用和结果沉淀。
- 人工审批：发布、自动回复、订单状态修改、导出客户数据等动作必须审批。
- 前端拆分：禁止继续把新增页面堆到 `public/app.js` 末尾。
- 文档与检查：改动后同步文档，并通过 `npm run check`。

## P1 高优先级

- 抖音真实平台闭环强化。
- 客服知识库 RAG 与依据引用。
- 视频异步生成队列、模型能力矩阵和成本统计。
- 操作审计日志。

## 非目标

- 不同时浅接所有平台。
- 不把模拟数据伪装成真实数据。
- 不让 AI 自动执行高风险动作。
- 不用花哨大屏替代清晰的业务状态。
