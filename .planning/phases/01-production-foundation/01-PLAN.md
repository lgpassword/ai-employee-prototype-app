# 阶段一计划：生产底座补强

## 目标

让现有原型先具备可恢复的业务状态，并为后续数据库、登录安全和 repository 分层迁移提供明确入口。

## 第一切片

1. 新增本地 JSON 快照持久化适配层。
2. 服务启动时加载快照，API 请求结束后保存快照。
3. 不持久化全局 `session`，避免重启后沿用旧登录态。
4. 增加持久化状态查询接口，便于人工确认运行状态。
5. 将新文件加入 `npm run check`。

## 后续切片

1. 增加密码哈希和真实 session/JWT。
2. 把用户、额度、视频草稿、客户会话拆到 repository。
3. 增加 API 输入校验和统一错误响应。
4. 增加基础测试，覆盖登录、额度、视频草稿和客户会话。

## 验收

- `npm run check` 通过。
- POST/配置类业务动作后生成 `.local/store.json`。
- 重启服务后用户、视频草稿、客户会话和设置数据从快照恢复。
